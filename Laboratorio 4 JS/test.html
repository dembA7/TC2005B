Preguntas

¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Java es un lenguaje de programación de propósito general que se utiliza principalmente para desarrollar aplicaciones empresariales de alta escala, mientras que JavaScript es un lenguaje de programación interpretado que se utiliza principalmente para desarrollar aplicaciones web interactivas.

¿Qué métodos tiene el objeto Date?

  • getDate(): devuelve el día del mes (entre 1 y 31) de la fecha especificada.
  • getDay(): devuelve el día de la semana (entre 0 y 6, donde 0 es domingo)
  • getFullYear(): devuelve el año (con cuatro dígitos) de la fecha especificada.
  • getHours(): devuelve la hora (entre 0 y 23) de la fecha especificada.
  • getMinutes(): devuelve los minutos (entre 0 y 59) de la fecha especificada.

¿Qué métodos tienen los arreglos?

  • concat(): fusiona dos o más arreglos en uno nuevo.
  • every(): comprueba si todos los elementos de un arreglo cumplen una condición.
  • filter(): crea un nuevo arreglo con todos los elementos que cumplan una condición determinada.
  • find(): devuelve el primer elemento que cumpla una condición determinada.
  • forEach(): ejecuta una función para cada elemento del arreglo.

¿Cómo se declara una variable con alcance local dentro de una función?

En JavaScript, una variable con alcance local dentro de una función se declara usando la palabra clave "let" o "const" dentro del cuerpo de la función.

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

  • Accesibilidad y modificabilidad: Las variables globales pueden ser accedidas y modificadas desde cualquier parte del código, lo que puede llevar a comportamientos inesperados y errores difíciles de depurar.
  • Colisiones de nombres: Las variables globales pueden tener nombres comunes que colisionen con nombres de variables locales o de otras funciones, lo que puede llevar a errores de lógica o a la sobreescritura accidental de valores.
  • Memoria y rendimiento: El uso excesivo de variables globales puede aumentar el consumo de memoria y afectar el rendimiento del código, especialmente en programas grandes y complejos.
  • Reusabilidad y encapsulamiento: El uso de variables globales puede dificultar la reutilización y el encapsulamiento de las funciones, ya que éstas dependerán de variables externas y no podrán ser usadas de manera autónoma.
